# 女娲完整蒸馏提示词模板

> 适用于：有人物本地语料（演讲/书籍/股东会纪要等），想做全面画像 + RAG 增强检索的蒸馏场景。
> 以芒格项目为参考范本——最终产出 8,834 行调研数据、75 个源文件、10 条决策启发式。

## 第一步：启动蒸馏（本地语料模式）

```

我有本地语料：'[语料目录绝对路径]'

要求：
1. 使用本地语料优先模式——先分析本地素材，网络搜索仅做补充
2. 保留所有搜索到的原始文本，不要只存摘要
3. 每个 Agent 的搜索 snippet 也写入文件
4. sources/ 目录下存入所有素材原文
5. Phase 1 跑完后暂停，等我确认调研质量再进入 Phase 2
```

## 第三步附加：多人对话转录稿处理（Speaker-Filtered Transcripts）

> ⚠️ 这一步专门处理会议/访谈/对谈等**多人对话**转录稿。单人演讲直接走普通 PDF/EPUB 流程即可。

**核心问题**：以伯克希尔股东大会为例，巴菲特发言时长占 80-90%，芒格只占 10-20%。如果把完整逐字稿直接扔进知识库，RAG 检索大概率抓取到的是**巴菲特的原话**，而不是芒格的。两人的投资哲学虽同源，但表达风格和具体偏好有差异——你的 Agent 会变成"巴菲特"，而不是"芒格"。

**解决方案**：写脚本做 Speaker Filtering + Context Binding。

```
现在处理多人对话转录稿。我需要只提取[目标人物]的发言，同时保留上下文。

【处理脚本要求】写一个 Python 脚本，完成以下步骤：

1. Speaker Filtering（说话人过滤）：
   - 解析转录稿中每一行，识别说话人标签（如 "Warren Buffett:" / "Charlie Munger:" / "Question:"）
   - 只提取目标人物（芒格）的发言段落
   - 丢弃其他所有人的发言正文

2. Context Binding（上下文绑定）：
   - 芒格最常说 "I have nothing to add" 或 "Warren's right"
   - 如果只提取这句话，脱离上下文 AI 完全无法使用
   - 每条芒格发言必须绑定：
     { "question": "[听众提问的原文/摘要]",
       "buffett_summary": "[巴菲特在芒格之前说的话的摘要——如果芒格是接着巴菲特说的]",
       "munger_response": "[芒格原话，一字不改]" }
   - 如果芒格是第一个回答的人（前面没有巴菲特）→ buffett_summary 为空

3. Filtering（无效语料剔除）：
   - 删除以下无意义回复（它们会污染向量空间）：
     * "I have nothing to add."
     * "I have nothing further to add."
     * "Warren's covered it."
     * 纯 "Yes." / "No." 的单字回复
     * 芒格只是在附和/重复巴菲特刚说过的话（语义去重）
   - 保留标准：芒格的发言必须包含独立的、有信息量的观点

4. Chunking（切片）：
   - 每条 (question + buffett_summary + munger_response) 作为一个独立 chunk
   - 输出为 JSON Lines 格式，方便后续导入 RAG
   - 文件存到 data/extracted/transcripts/munger_only_qa.jsonl

5. 输出示例：
   {"question": "How do you view the current inflation environment?",
    "buffett_summary": "Buffett explained that inflation acts as a tax on fixed-income investors and that Berkshire's businesses with pricing power can pass through costs.",
    "munger_response": "I think we're seeing the inevitable consequence of what happens when you print ungodly amounts of money. I'm not sure exactly where it goes from here, but I'm sure the purchasing power of the dollar will decline over time. That's been true for a long time.",
    "year": 2022,
    "source": "Berkshire Hathaway Annual Meeting 2022"}

【后续处理】
- 将 JSON Lines 文件中的高质量条目（芒格有实质发言的）追加到 07-rag-quote-database.md
- 标注来源 [BRK-年份]
- 更新 08-topic-taxonomy.md 的源文件清单
```

## 第四步：导入核心书籍（提升数据质量的关键步骤）

这一步区分两类书籍的不同角色，分别处理。

### 4A：核心理论基石——《穷查理宝典》类

这类书籍是人物亲笔或授权编纂的**系统理论著作**，包含经过修订的定稿版核心文章。它们应该成为知识库的**理论骨架**。

```
现在导入核心理论著作，提升数据质量。

我有《[书名]》电子版：'[epub文件路径]'

这本书是[人物]的[定位描述，如：授权编纂的系统理论著作]。
请按以下要求处理：

【角色定位】这本书是知识库的「理论基石」，优先级高于所有零散演讲和股东会纪要。
- 书中的文章是作者本人修订的定稿版，比原始演讲更权威
- 编辑框架本身也有信息价值——它揭示了作者思维的系统结构

【提取步骤】
1. 安装提取库：pip3 install ebooklib beautifulsoup4
2. 提取全部章节为 .txt，输出到 data/extracted/[书名]/
3. 同时生成 00-FULL-BOOK.txt 方便全文检索
4. 复制到 sources/books/ 目录

【重点处理内容】
- 特别注意书中经过作者修订的核心章节（如芒格《人类误判心理学》2005年定稿版，将三次演讲合并重写）
  → 用这个版本完全替换 01-writings.md 中基于早期演讲/零散记录的碎片化引用
  → 先完整提取原文确保不丢失，后续第五步会统一做结构化压缩（保留原话定义+1-2个典型案例+应对策略，压缩到 200-500 字）
- 编辑的框架性内容（如投资检查清单、思维模型分类体系）提取到 01-writings.md
- 新发现的金句和原文段落追加到 07-rag-quote-database.md，标注 [BOOK-ABBR] 来源
- 年谱/生平信息补充到 06-timeline.md
- 更新 08-topic-taxonomy.md 的源文件清单，标注为「权威定稿」质量等级

【更新范围】01、02、05、06、07、08 号文件
【来源标记】所有新增内容标注 [PCA]（或其他书籍缩写，如 [TAO]）
```

### 4B：实战案例库——《芒格之道》类

这类书籍是**按时间编排的对话实录**，经过文字净化。它们应该成为 Agent 回答具体问题时的**参考范例库**。

```
现在导入实战案例库，提升数据质量。

我有《[书名]》电子版：'[epub文件路径]'

这本书是[人物]的[定位描述，如：1987-2022年股东会Q&A实录]。
请按以下要求处理：

【角色定位】这本书是知识库的「实战案例库」，用于回答具体商业/宏观问题。
- 内容是按年份编排的真实Q&A对话，经过文字净化（去除口语冗余）
- 填补了零散会议纪要无法覆盖的长时间跨度
- 对话形式天然适合 RAG 检索——用户问具体问题，这里找相似场景的原文回答

【提取步骤】
1. 安装提取库：pip3 install ebooklib beautifulsoup4
2. 提取全部章节为 .txt，输出到 data/extracted/[书名]/
3. 每章按年份命名（如 1987-西科股东会.txt），方便按时间检索
4. 同时生成 00-FULL-BOOK.txt
5. 复制到 sources/books/ 目录

【重点处理内容】
- 优先提取人物早年（如前 10 年）和最近 2-3 年的时间段内容
  （这些通常是已有多数零散记录的盲区）
- 提取 30-40 条高质量Q&A原文追加到 07-rag-quote-database.md
  → 每条标注 [缩写-年份] 来源（如芒格之道用 [MT-2021]）
  → 重点标签根据人物领域调整：#investing #business #life-wisdom 等
- 识别跨年复现的新论点（同一个观点在多个年份出现 = 真信念）
  → 追加到 01-writings.md 的「反复出现的核心论点」部分
- 识别立场演化轨迹（如从乐观到悲观、从回避到接受）
  → 追加到 02-conversations.md 和 05-decisions.md
- 填补 06-timeline.md 的时间线空白
- 更新 08-topic-taxonomy.md，标注「已解决」的缺口

【更新范围】01、02、05、06、07、08 号文件
【来源标记】所有新增内容标注 [缩写-年份]（如 [MT-2021]）
```

### 4C：两本书的协同效应

导入完成后，运行一次验证确认互补关系：

```
现在验证两本书的协同效应：

1. 检查理论书的框架是否完整覆盖了核心理论体系
   - 每一条都应该先完整提取原文，再按第五步压缩到 200-500 字
   - 如果某条还是碎片 → 回到定稿版原文补全后再压缩

2. 检查案例书是否填补了时间线缺口
   - 人物早年（前 10 年）是否有足够的 Q&A 内容
   - 最近 2-3 年是否覆盖
   - 如果仍有空白年份 → 标注在 08-topic-taxonomy.md 中

3. 两本书的重叠话题做交叉验证：
   - 同一观点在两本书中都出现 → 标注为「高可信度」
   - 同一观点在书A有、书B没有 → 标注为「可参考」
   - 两本书有矛盾 → 优先采信定稿版理论书，标注差异
```

## 第五步：结构化精简——高密度保留原意

> ⚠️ 这是全文通用原则，不是只针对心理学部分。

原始素材提取后会存在大量冗余：同一个观点演讲中重复三遍、过渡叙述、与核心论点无关的轶事。直接保留全部会撑爆 token 预算。但用 AI 总结又会歪曲原意。

解决方案：**结构化萃取**，不是 AI 总结。

```
现在对所有调研文件做结构化精简。全文适用。

【核心原则】保留[人物名]的原话，压缩冗余，不改变原意。

【操作方法】每条重要内容只保留三个槽位：
1. 本质定义 —— [人物名]怎么描述这个观点？用原话，一字不改
2. 典型例证 —— 精选 1-2 个最有代表性的案例，不是全部案例
3. 应对策略 —— [人物名]建议怎么做？（如果原文有）

【可以删除的】
- 同一观点为了演讲效果重复的段落（保留最清晰的那一版）
- 纯过渡性叙述（"接下来我要讲..."）
- 与核心论点无关的轶事
- 重复出现的相同案例（同一个故事讲了三次，保留信息最全的那次）

【必须保留的（一字不改）】
- 自创术语（如芒格的 Lollapalooza 效应、铁锤人倾向、咬手指错误）
- 金句和定义句（这些是表达 DNA 的核心）
- 跨学科类比（铁路/赛马/生态系统等比喻）
- 与其他概念/模型的交叉引用
- 数字、日期、具体金额

【压缩目标】每条从 500-5000 字压缩到 200-500 字。

【自检标准】拿着精简版和完整版回答同一个用户问题，结论一致 = 通过。结论偏移 = 删多了，恢复。

【适用范围】所有 01-08 号调研文件、RAG 语料库、SKILL.md 中的引用段落。
```

## 数据质量检查清单

发布前确认以下每一项：

### 结构化精简（全文适用）
- [ ] 所有长文条目已做三槽位萃取（本质定义 + 典型例证 + 应对策略）
- [ ] 芒格原话定义句一字未改
- [ ] 自创术语、金句、数字、交叉引用全部保留
- [ ] 每条精简后自检：和完整版回答同一问题，结论一致

### 理论层（定稿版系统著作贡献）
- [ ] 核心理论体系已先完整提取原文，再按三槽位压缩到 200-500 字（不是碎片摘要，也不是 5000 字原文堆砌）
- [ ] 核心概念有精确定义（如物理学比喻级别），不是泛泛介绍
- [ ] 框架性工具完整收录（如检查清单、分类体系 + 细则）
- [ ] 人物年谱精确到具体日期和金额
- [ ] 来源标记清晰（如 [PCA]）

## 关键踩坑记录

1. **调研 Agent 会偷懒缩写**：即使要求"保留完整原文"，Agent 也可能用 `...` 截断。Phase 1 完成后必须扫描所有 `...` 并比对原文补全。

2. **PDF 要提前提取**：Phase 1 启动前就应该把 PDF 全部转 txt，否则 Agent 读不到 PDF 内容。

3. **中文翻译 ≠ 原文**：很多中文翻译会丢失人物的讽刺语气和语域混合。英文原版能拿到的一定要拿。

4. **标准女娲不包含 RAG 和书籍导入**：07/08 文件和书籍处理流程必须额外要求。

5. **电子书版本比单篇演讲更权威**：书里的版本经过了作者修订（如 PCA 的 2005 年定稿版心理学），应优先采信并替换基于早期演讲的碎片化引用。

6. **两本书的角色不同，处理方式不同**：理论书要「深」——每条理论挖到完整论文级别。案例书要「广」——覆盖尽可能多的时间段和话题。

7. **检查点不能省略**：Phase 4 和 Phase 5 是质量的下限保证。跳过它们交付的东西可能看起来像但经不起推敲。

8. **Phase 2 要重跑**：导入大量新书后，原来提炼的模型可能不完整。必须重新审视是否有新模型或新启发式浮现。

9. **完整 ≠ 有用，压缩 ≠ 丢失**：3000 字的原文对 RAG 检索是负担。结构化萃取（保留原话定义 + 1-2 个例子 + 对策）既降 token 又不歪曲原意。AI 总结会引入偏差，结构化不会。全文适用，不只心理学。

10. **多人对话转录稿必须先做 Speaker Filtering**：直接把会议逐字稿扔进知识库，模型会变成占发言时长 80-90% 的那个人。必须写脚本只提取目标人物的发言，并绑定提问者的问题和其他人的发言摘要作为上下文元数据。同时剔除 "I have nothing to add" 等无意义回复，避免污染向量空间。
